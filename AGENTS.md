# AGENTS.md

## 项目目标

本项目当前 MVP 是 `focus-reading-launcher`，中文可描述为“专注阅读启动器”。目标是把用户提供的网页链接、粘贴文本或本地文件，转成一次有时间限制、有阅读进度、有进入仪式、有退出仪式的阅读 Session。

它不是普通 todo list，也不是复杂知识库；它应当像一个安静的学习仪式工具，帮助用户在图书馆、书店或低干扰环境中完成一次明确、有边界的阅读。

## 用户使用场景

用户可能会阅读 GPT 周报、行业文章、课程材料、会议资料、Markdown 笔记或本地文档。输出和界面要帮助用户判断：这次读什么、读多久、读到哪里、带走什么，而不是鼓励无限收藏或无目标刷材料。

## 输出语言要求

- 用户可见内容必须用中文。
- 代码、字段名、目录名、文件名必须用英文。
- README、网页文案、模板说明要优先服务“低干扰、有边界、有收束感”的阅读体验。

## 文件命名规范

- 目录名、脚本名、字段名使用英文小写和下划线或连字符。
- 静态网页入口优先放在 `web/`。
- 私人输出默认不要提交到仓库；如未来新增导出目录，应通过 `.gitignore` 避免提交用户材料。

## 隐私和安全边界

- 第一版不引入后端、数据库、账号登录、付费系统、分析 SDK 或 API key。
- 默认只使用用户主动输入或上传的材料。
- 不自动抓取网页正文，不做违反网站规则的爬虫。
- 不自动上传用户材料。
- 不自动发送邮件、微信或任何跟进消息。
- `localStorage` 只用于本地保存 Session 草稿和历史记录。
- PDF 解析第一版只保留占位提示，除非用户明确要求，不要加入复杂解析依赖。

## 产品原则

- 必须保留三个阶段：进入仪式、专注阅读、退出仪式。
- 开始阅读前不能直接进入内容，必须先出现进入仪式。
- 退出时要收束为“读到最重要的点”。
- 不要把应用做成普通任务清单、收藏夹或泛泛效率工具。
- 界面应极简、安静、低干扰，按钮清晰，适合展示 demo。

## 代码原则

- 优先使用原生 HTML/CSS/JavaScript。
- 第一版保持静态网页，可直接本地打开或部署到 GitHub Pages。
- 代码尽量简单、可读、本地优先。
- 不要在 import 周围写 try/catch。
- 如未来引入构建工具或框架，必须先说明原因，并保证部署路径清晰。
