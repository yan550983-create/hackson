# 产品规格：活动情报与低能耗社交助理

## 产品定位

`event-networking-assistant` 是一个本地优先的活动准备与复盘工具。它将线下活动视为“信息场”和“关系场”，帮助用户在参加活动前判断场域结构、重点角色和低能耗行动，在活动后沉淀有效信息、跟进对象和人物记忆。

它不是泛社交产品，也不是复杂 CRM。v0 以 Codex Skill、Markdown 模板、Python 脚本和静态网页 demo 为核心。

## 用户场景

用户经常参加 AI、出海、TikTok Shop、跨境电商、创业、投资、黑客松、青年科创和行业分享类活动。用户关注的不是认识更多人，而是：

- 判断这个活动是什么局。
- 判断主办方真正想聚集什么人。
- 判断哪些嘉宾、机构或角色值得聊。
- 明确可以聊什么和该问什么。
- 在低精力状态下完成最小有效行动。
- 会后整理信息、识别噪音、生成跟进任务。

## 核心工作流

### 阶段 A：活动前准备

1. 用户填写 YAML 活动信息或网页表单。
2. 工具生成会前简报。
3. 简报区分确定信息、推测判断、待确认信息。
4. 输出可聊对象、问题清单、观察信号、避免话题和跟进预案。

### 阶段 B：活动后复盘

1. 用户粘贴活动后笔记或转写文本。
2. 工具生成结构化复盘草稿。
3. 输出人物、对话摘要、价值信息、噪音提醒、跟进优先级和话术草稿。
4. 不自动发送任何消息。

## v0 功能

- Codex Skill 说明。
- 本地 Markdown 模板。
- YAML 活动信息模板。
- 虚构示例活动与虚构转写。
- Python 会前简报生成脚本。
- Python 转写复盘生成脚本。
- 原生 HTML/CSS/JavaScript 静态网页 demo。
- GitHub Pages 部署工作流。
- 隐私边界文档。

## v1 路线图

- 支持粘贴活动链接后手动整理信息。
- 支持从用户提供的网页文本中提取活动信息。
- 支持更完整的人物卡片。
- 支持多活动管理。
- 支持把会前简报和会后复盘关联起来。

## v2 路线图

- 支持联网搜索公开资料。
- 支持嘉宾公开资料整理。
- 支持活动主办方历史活动分析。
- 支持导出跟进提醒。
- 支持本地 Whisper 或 API 转写。
- 支持基于用户上传海报 OCR 后整理活动信息。

## v3 路线图

- 做轻量 Web UI。
- 支持 SQLite 本地数据库。
- 支持联系人关系沉淀。
- 支持日历集成。
- 支持邮件/微信跟进草稿。
- 支持多活动复盘库。

## v4 路线图

- 做真正 App 或插件。
- 接入 Google Calendar、Gmail、Notion/飞书。
- 支持定期活动准备提醒。
- 支持多活动关系网络视图。
- 支持个人长期关系资产沉淀。

## 非目标功能

v0 不做复杂 App、不做后端、不做数据库、不做登录、不做真实爬虫、不做真实音频转写、不接 API key、不引入 React/Vue/Vite、不做联系人自动同步、不自动发送邮件或微信。

## 风险边界

- 只处理公开信息和用户明确提供的材料。
- 不推断私人隐私、敏感身份、声纹或人脸信息。
- 所有可能到场人群必须标注为推测。
- 所有事实必须尽量附来源，缺来源标注“来源待补充”。
- 普通寒暄不应被过度解读为人脉价值。
- 输出跟进话术草稿，但不自动发送。

## 为什么先做 Skill + 静态 demo，而不是完整 App

第一版的核心风险不是工程复杂度，而是判断框架是否清晰、隐私边界是否清楚、用户是否真的需要“低能耗但高信息判断力”的活动工具。Skill + 静态 demo 可以最快验证工作流、模板和输出结构，并避免过早引入后端、数据库、登录和隐私风险。
